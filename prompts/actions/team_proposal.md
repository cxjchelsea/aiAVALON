# 队伍提议 (Team Proposal) Prompt模板

## 任务描述
作为队长，你需要提议一个队伍去执行任务。队伍人数由当前任务配置决定。

## 思考步骤（Chain-of-Thought）

请按照以下步骤进行思考，并在回答中展示你的思考过程：

### 步骤1：分析当前游戏状态
- 当前是第几轮任务？
- 成功/失败任务数是多少？
- 投票轮次是多少？（注意：如果投票轮次>=4，这是第5次投票，必须通过，否则坏人直接获胜）

### 步骤2：分析任务历史（关键推理依据）
**重要：如果这是第1轮，没有任务历史！**
- 如果是第1轮：没有历史信息，只能基于可见信息、信念系统和角色能力判断
- 如果有历史：仔细分析之前的任务结果：
  - **关键推理规则1**：如果任务失败且失败票数等于队伍人数，说明队伍中**所有人都是坏人**！
  - **关键推理规则2**：如果任务失败但失败票数小于队伍人数，说明队伍中有部分坏人在破坏任务
  - **关键推理规则3**：如果任务成功，说明队伍中可能都是好人（或者坏人选择隐藏身份）

### 步骤3：评估每个玩家
基于以下信息评估每个玩家：
- 信念系统中的信任度评分
- 之前的投票行为
- 之前的任务投票行为
- 发言内容
- 可见信息（如果你有特殊能力）

### 步骤4：考虑角色目标
- 如果你是好人：选择最可信的玩家
- 如果你是坏人：尽量包含坏人队友，但也要包含一些好人以隐藏身份

### 步骤5：做出决策
基于以上分析，选择最合适的队伍成员。

## 输出格式

请以JSON格式返回你的决策：

```json
{
    "thinking_process": {
        "step1_game_state": "你的分析...",
        "step2_mission_history": "你的分析...",
        "step3_player_evaluation": "你的分析...",
        "step4_role_objectives": "你的分析...",
        "step5_decision": "你的最终决策理由..."
    },
    "team": [玩家ID列表，例如 [0, 1, 2]]
}
```

## 重要提醒
- **第一轮特别提醒**：如果这是第1轮任务，游戏刚刚开始，**没有任务历史**。不要提及"前几轮"、"之前的表现"等不存在的信息！
- 你的回答必须基于提供的游戏事实，不得编造信息
- 如果任务历史显示某个队伍全部是坏人，必须避免再次选择相同的组合
- 考虑流局风险：如果这是第5次投票，必须确保队伍能被通过
